# Lesson 1: Basic RAG — Building End-to-End

## Learning Objectives
By the end of this lesson, you'll understand:
- How to load documents and chunk them
- Why naive retrieval fails sometimes
- The quality jump with semantic search

## What You'll Build
A complete simple RAG system that:
- Stores documents with their embeddings in a vector database.
- Loads documents from disk into a ChromaDB vector store.
- Converts text documents into embeddings using SentenceTransformer.
- Orchestrates the retrieval and uses an LLM to generate answers to queries.

## Notebooks
- `simple_rag.ipynb` — Simple RAG implementation: from document loading and chunking to retrieval and answering. Naive RAG vs. better RAG (HyDE example).

- `simple_rag_answered.ipynb` - Answered notebook. 

## Ready to begin? 
Open the notebook `simple_rag.ipynb` and begin to learn!  Remember that the best way to learn is trying, failing and implementing a working version yourself.
